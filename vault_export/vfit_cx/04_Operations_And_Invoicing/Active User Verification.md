# Active User Verification

Process for verifying active-user counts before billing cycles and during client disputes.

---

## What Is An Active User?

An active user is an employee who, within the billing period:

- logged into Vantage Fit at least once, and
- recorded at least one qualifying activity

**Important:** the exact definition may vary by contract. Always confirm the signed commercial definition before billing.

---

## Standard Verification Process

### Step 1 - Request the Count

Submit the request to the data team with:

- client company name
- exact billing period
- exact definition of "active"
- any exclusions such as test accounts

See [[05_Internal_Collaboration/Data Team Coordination]] for the request format.

### Step 2 - Review the Report

Check for:

- total active-user count
- weekly breakdown if the month had partial rollout or shutdown periods
- anomalies or large swings
- duplicate or internal test accounts

### Step 3 - Cross-Reference Against Agreement

- Compare actual count vs contracted seats
- Check minimum billing thresholds
- Check overage handling
- Apply pricing tiers or discounts if relevant

### Step 4 - Prepare Client-Facing Summary If Needed

If the client asks for detail, be ready to explain:

- the date range used
- the exact active-user definition
- whether the count reflects login plus activity
- whether test accounts were excluded

Share a summary view unless a deeper approved review is required.

---

## Related Counts That Often Matter

When a billing dispute happens, these counts often help explain the gap:

- invited users
- registered users
- active users
- challenge participants
- duplicate or disabled accounts

This helps separate "seats available" from "people who actually used the platform."

---

## Verification For Client Disputes

1. Acknowledge the dispute the same business day
2. Request a re-pull for the disputed period
3. Reconfirm the contract definition before replying
4. Share the methodology and summary data with the client
5. If a discrepancy is confirmed, escalate for approval before adjusting billing
6. Log all communication in CRM or account notes

---

## Common Issues

| Issue | Action |
|---|---|
| Test accounts inflate count | Exclude them before final billing summary |
| Duplicate users appear twice | Ask the data team to deduplicate |
| Client HR roster does not match platform users | Check upload timing and registration gaps |
| Partial month after launch | Review whether contract allows prorated treatment |
| Holiday or shutdown month | Check whether the agreement addresses reduced activity periods |

> Hi [Data Team Contact],
>
> Could you please pull the active-user count for **[Company Name]** for **[Start Date] to [End Date]**?
>
> Per the contract, active is defined as: [definition].
>
> Please exclude test accounts and flag any anomalies.
>
> Thank you.
