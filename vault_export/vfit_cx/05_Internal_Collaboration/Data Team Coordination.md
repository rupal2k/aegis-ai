# Data Team Coordination

Process for working with the data team on reporting, active-user verification, and billing support.

---

## When To Involve The Data Team

| Request | Data team? |
|---|---|
| Active-user count for billing | Yes |
| Campaign participation report | Yes |
| Client-specific engagement analytics | Yes |
| Department breakdown | Yes |
| Wellness trend summary | Yes |
| Invoice discrepancy investigation | Yes |
| Standard challenge setup | No |
| Standard email template | No |

---

## Standard Data Request Process

### Step 1 - Prepare The Request

Have these ready:

- exact client company name
- exact date range
- specific metric or report needed
- business purpose
- due date
- exclusions or custom rules

### Step 2 - Submit The Request

- Use the agreed Slack channel, form, or workflow
- State urgency clearly
- Mention if the output is client-facing

### Step 3 - Review Before Use

When data comes back:

- verify the time period
- check for anomalies
- confirm the requested definitions were used
- make sure test accounts were handled correctly

---

## Most Useful Report Menu For CX

| Need | Ask for |
|---|---|
| Billing validation | Active-user count with contract definition |
| Program review | Engagement trend summary |
| Post-challenge recap | Participation plus result summary |
| Leadership visibility | Department breakdown |
| Renewal prep | Usage trend plus headline outcomes |

---

## Data Handling Rules

- Aggregated views are the safest default for HR stakeholders
- Sensitive health detail should not be shared casually
- If a metric may affect billing, include the contract definition in the request
- If a metric may be reused externally, verify it against the approved proof-point note first

---

## Standard Timing

- Billing-related requests: submit early in the month
- Standard challenge reports: allow 3 to 5 business days
- Complex or custom requests: allow 5 or more business days
