# Admin Setup Guide

A step-by-step reference for configuring a new client's admin environment on Vantage Fit.

---

## Step 1 - Access the Admin Panel

1. Log into the Vantage Fit admin portal
2. Navigate to **Company Management -> Add New Company**
3. Enter company details:
   - Company name
   - Industry
   - Employee count
   - Subscription plan
   - Billing start date

---

## Step 2 - Configure Company Settings

| Setting | Action |
|---|---|
| Company logo | Upload client logo (PNG or SVG, square preferred) |
| Timezone | Set to the client's primary timezone |
| Language | Default to English unless otherwise agreed |
| Points and rewards | Confirm whether points, badges, and marketplace rewards are in scope |
| Leaderboard | Enable or limit based on client preference |
| Team features | Enable for clients running team-based challenges |
| Notifications | Confirm reminder cadence for admins and employees |
| Reports and visibility | Confirm who needs admin access and reporting rights |

---

## Step 3 - Create Admin User

1. Go to **Users -> Add User**
2. Enter admin full name and work email
3. Assign role: **Company Admin**
4. Set a temporary password per policy
5. Enable the account
6. Trigger welcome or credentials email

---

## Step 4 - Employee Data Upload

### Option A - CSV Upload
1. Download the employee upload template
2. Share the template with the client's HR/admin owner
3. Review the completed CSV for formatting issues
4. Upload via **Users -> Bulk Upload**
5. Review the upload report for errors

### Option B - Self-Registration
1. Enable self-registration in company settings
2. Share the registration link and company code
3. Monitor registration count during the first week

### Option C - HRIS / Managed Sync
1. Confirm the client's expected roster source
2. Align with the relevant internal integration or product owner if setup help is needed
3. Test with a limited group before relying on a full sync

---

## Step 5 - Confirm Program Foundations

Before handoff, align on the basics of the live program:

- First challenge type and launch timing
- Whether employees will rely on phone tracking only or connect devices
- Whether rewards are symbolic, points-based, or gift-card based
- Which admin will own reporting requests
- What success metric matters most in month one

Device-sync conversations should be especially clear if the client expects employees to use wearables such as Apple Watch, Fitbit, or Garmin.

---

## Step 6 - Notifications and Communications

- Enable admin summary emails if part of the setup
- Confirm challenge reminders and employee notifications
- Test a sample notification if the workflow allows it
- Share welcome and onboarding templates with the client admin

---

## Step 7 - QC Before Handoff

- [ ] Company profile is complete and accurate
- [ ] Admin account is active and accessible
- [ ] Employee data uploaded correctly
- [ ] At least one challenge or template is ready
- [ ] Rewards and leaderboard settings match the agreed program design
- [ ] Reporting owner is known
- [ ] Test login is successful

---

## Common Issues

| Issue | Fix |
|---|---|
| Admin cannot log in | Verify email, reset password, confirm account is active |
| Employee CSV upload fails | Check required fields, duplicate emails, and invalid formats |
| Logo is not displaying | Verify image format and dimensions |
| Timezone is incorrect | Update company settings and recheck time-based items |
| Device-sync expectations are unclear | Clarify employee guidance before launch |
